发散思维

最近编辑于 Jan 25, 2024

    真正的灵感的敌人是“专注”，因为这个时候你是聚合思维，聚合思维不适合发散寻找灵感。

    创造性思维的本质是两个距离比较远的节点结合出来，碰撞出的火花。

    所以你想要寻找灵感的时候，要让自己放松，让概念的范围扩展，用各种方式接触新的方式和事物，最后把想到的点子写出来，一起组合

    # 产生创意的七个原则

    # Capture捕获

    好的灵感不是刻意得来的
    我们要做的就是尽量留住那稍纵即逝的灵感。
    做练习

    1. 每天发现 1-3 个有趣的细节，比如某个有趣的招牌，大楼设计的某个细节
    2. 每天记录 1-3 个灵感，一个关键词，一个问题，一个联想
      不用花太久，每天几分钟就行

    # Maintain 保持

    敏感化技术：不一定和专注相悖，你需要先专注的思考一个问题，然后切换到后台，与你放松时候接触到的事物、影像结合起来，都可能与你的问题产生碰撞。

    L先生回建一个 Focus 清单，记录一些问题，在散步的时候提取几个问题放入后台

    # Insight洞察

    问题背后的本质是什么？
    我们把注意力集中在输入和输出，把中间的处理看成一个黑箱。然后去想有那些方法可以帮助系统从最开始的状态达到最终的状态。

    # Negative否定

    推翻既定常识/假设，会发生什么

    • 在一个系统把原本有的事情去掉
    • A到 B => B -A
    • 把一个状态极端的放大或者缩小
      • 比如手机续航，极端的只打电话待机一个月

    # Breakdown分解

    尽可能的写出相关的属性，分别列成两列，然后对两边的属性相加结合起来

    • 描述属性
    • 功能属性
    • 操作属性

    可以把之前记录下来的灵感，重新拆解组合

    # Order顺序

    按一定顺序/提问

    比如 6F 提问法

    第一次

    • Fact ：查询一些可以查到的事实数据
    • Feeling： 描述对其的感觉
      第二次
    • Feature：写下优点
    • Fears：写下缺点
      第三次
    • Foward，综合前面总结出他可能怎么做
    • Finsh，决定策略和行动

    # Exchange交换

    从研究对象上深入思考，其中哪些部件和流程是否可以替换。

    # SCAMPER模型/奔驰创意法

    • Substitute替换：能否将某个部件替换为别的？
    • Combine整合：哪些部分可以重新整合到一起？
    • Adjust调整：哪些部分有微调的空间？（小改)
    • Modify修改：材料、外观、功能等，能够修改？（大改）
    • Put to another use 另外用途：某些部分能否找到别的用途？
    • Eliminate 消除：哪些部件、流程可以去除？
    • Rearrange 重组：能够改变其结构、顺序？

    # 题目

    任选一个主题，用SCAMPER模型进行分析和优化。
    (以下题目2 选1即可)
    1)任选一家店铺，可以具体到某一家，也可以是某一类（火锅店，西餐店，咖啡店.…)。目的是提高收入。
    2)任选一款你熟悉的产品，种类不限。目的不限。（提示一下： 没有灵感的同学，可以拿一款你熟悉的APP进行分析优化，应该会比较有感触)

    B 站：

    1. 整合：追番和追剧可以合并，日常其实只看自己追的有没更新
    2. 替换： B 站记笔记的功能可以参考 语雀小记，记录碎片化的知识，最后能提供 API
    3. 调整：PC端的页面可以做成下拉更新，能增加人的留存时间（write 插件就这事
    4. 消除：会员购隐藏到后台；，采用联动去购买感觉更好(虽然是重要的盈利组成)
    5. 引入：可以引入 AI总结的功能，先让观众了解内容（每个视频总结一百年就好
